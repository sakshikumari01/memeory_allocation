📚 PAGING Definition

Paging: Memory management technique dividing physical RAM into fixed-size FRAMES and logical memory into PAGES.

KEY CONCEPTS:
• PAGE: Fixed-size logical memory block (e.g., 4KB, 8KB)
• FRAME: Fixed-size physical RAM block
• PAGE FAULT: Required page not in memory → Slow disk access
• PAGE HIT: Page already in memory → Fast access
• ALGORITHMS: FIFO (Simple), LRU (Better), OPTIMAL (Theoretical Best)

📚 Segmentation Concepts

SEGMENTATION: Memory divided into variable-size logical blocks (SEGMENTS)

1️⃣ BASE ADDRESS: Physical starting address of segment
2️⃣ LIMIT: Maximum size of segment in bytes (prevents illegal access)
3️⃣ OFFSET: Distance within segment from base
• ✅ Valid if Offset < Limit   ❌ Fault if Offset ≥ Limit

ADDRESS TRANSLATION: Physical Address = Base + Offset

🧮 Virtual Memory

VIRTUAL MEMORY: A memory management technique that gives an application the illusion of a very large, contiguous main memory by combining RAM with disk-based secondary storage (swap space). The OS maps virtual addresses used by a program to physical addresses in RAM, loading pages on demand.

🔹 Why Virtual Memory?

• Run programs larger than physical RAM
• Each process gets its own isolated address space (security + safety)
• Efficient sharing of memory between processes
• Simplifies linking, loading, and relocation

🔹 Key Terms

• Virtual Address: Address generated by the CPU (per-process).
• Physical Address: Actual location in RAM.
• Page Table: Mapping from virtual pages to physical frames.
• MMU: Hardware that performs the translation.
• TLB: Cache of recent page-table entries for fast translation.
• Swap Space: Area on disk used to hold pages evicted from RAM.

🔹 Demand Paging

Pages are brought into RAM only when needed. If a referenced page is not present, a page fault occurs and the OS loads it from disk, possibly replacing another page using FIFO / LRU / OPTIMAL.

🔹 Address Translation Flow

1. CPU emits a virtual address → split into page number + offset.
2. MMU checks the TLB; on miss it walks the page table.
3. If the page is in RAM → form physical address = frame × pageSize + offset.
4. If not → page fault → OS loads page from disk → retry instruction.

🔹 Thrashing

When a system spends more time swapping pages in/out than executing useful work. Caused by too many active processes vs. available frames. Mitigated by working-set models and load control.

🔹 Effective Access Time (EAT)

EAT = (1 − p) × memory_access + p × page_fault_time
where p = page fault rate. Even small p hurts performance because page fault time is huge compared to RAM access.

🔹 Advantages vs Disadvantages

✅ Advantages: Larger logical address space, isolation, multiprogramming, on-demand loading.
❌ Disadvantages: Page fault overhead, thrashing risk, hardware (MMU/TLB) complexity.
